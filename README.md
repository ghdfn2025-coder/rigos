# 리고스 익명 봇

Discord 서버에서 `/익명`으로 메시지를 보내면 지정된 채널에는 **리고스 봇의 메시지로만 게시**되고, 실제 작성자·원문·작성 시각은 서버 소유자 전용 로그 채널과 PostgreSQL에 저장됩니다.

## 명령어

### 모든 멤버

- `/익명 내용:<메시지>`  
  `///`는 줄바꿈으로 변환됩니다.

### 서버 소유자 전용

- `/익명관리 설정`
- `/익명관리 조회`
- `/익명관리 상태`
- `/익명관리 통계`
- `/익명관리 로그정리`

관리 명령어는 코드에서도 서버 소유자 ID를 확인하므로 다른 관리자가 실행할 수 없습니다.

> Discord의 명령어 표시 권한은 서버의 역할·사용자별 통합 설정 영향을 받습니다. 코드만으로 “서버 소유자 화면에만 100% 노출”을 보장하기는 어렵습니다. 아래의 **관리 명령어 숨기기** 절차까지 적용하세요.

---

## 1. Discord 애플리케이션 만들기

1. Discord Developer Portal에서 **New Application**을 누릅니다.
2. 이름을 `리고스`로 정합니다.
3. 왼쪽 **Bot** 메뉴에서 봇을 생성합니다.
4. **Reset Token**으로 토큰을 발급하고 안전하게 보관합니다.
5. 이 봇은 일반 채팅 내용을 읽지 않으므로 `Message Content Intent`는 켜지 않아도 됩니다.

## 2. 봇 초대하기

Developer Portal의 **OAuth2 → URL Generator**에서 다음 Scope를 선택합니다.

- `bot`
- `applications.commands`

권장 Bot Permissions:

- View Channels
- Send Messages
- Embed Links
- Read Message History

메시지 삭제 기능을 사용하지 않으므로 `Manage Messages`와 `Administrator`는 필요하지 않습니다.

생성된 URL로 봇을 서버에 초대합니다.

## 3. PostgreSQL 준비

Railway 기준:

1. 새 프로젝트를 만듭니다.
2. `+ New` → `Database` → `PostgreSQL`을 추가합니다.
3. PostgreSQL 서비스의 `DATABASE_URL` 변수를 사용합니다.
4. GitHub에 이 폴더를 올리고 Railway에 저장소를 연결합니다.
5. 봇 서비스 Variables에 다음을 등록합니다.

```env
DISCORD_TOKEN=봇_토큰
DATABASE_URL=${{Postgres.DATABASE_URL}}
DEV_GUILD_ID=테스트할_서버_ID
```

`DEV_GUILD_ID`를 넣으면 해당 서버에 명령어가 빠르게 반영됩니다. 완성 후 전역 배포하려면 값을 지우고 재배포할 수 있습니다.

## 4. 로컬 실행

Python 3.11 이상 권장.

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

macOS/Linux:

```bash
source .venv/bin/activate
```

설치:

```bash
pip install -r requirements.txt
```

`.env.example`을 `.env`로 복사하고 값을 입력한 뒤:

```bash
python bot.py
```

## 5. Discord 채널 만들기

예시:

- `#리고스-작성`
- `#리고스`
- `#리고스-로그`

### `#리고스`

- 일반 멤버: 메시지 보내기 거부
- 봇: 채널 보기, 메시지 보내기 허용

### `#리고스-로그`

- `@everyone`: 채널 보기 거부
- 서버 소유자: 채널 보기 허용
- 리고스 봇: 채널 보기, 메시지 보내기, 임베드 링크 허용

서버에서 `Administrator` 권한을 가진 다른 사람은 숨겨진 채널을 볼 수 있으므로, 정말 소유자만 봐야 한다면 다른 관리자에게 Administrator 권한을 주지 않아야 합니다.

## 6. 최초 설정

서버 소유자가 다음 명령어를 실행합니다.

```text
/익명관리 설정 게시채널:#리고스 로그채널:#리고스-로그 명령채널:#리고스-작성 쿨타임:60 최대글자수:1000
```

확인:

```text
/익명관리 상태
```

이제 멤버는 다음처럼 사용합니다.

```text
/익명 내용:오늘 있었던 일입니다.///정말 기뻤어요.
```

공개 채널:

```text
오늘 있었던 일입니다.
정말 기뻤어요.
```

사용자에게만 보이는 응답:

```text
✦ 리고스를 통해 익명 메시지가 전송되었습니다.

다른 이용자에게는 작성자 정보가 표시되지 않습니다.
운영을 위해 서버 소유자 전용 로그에 작성 기록이 보관됩니다.
```

## 7. 관리 명령어를 소유자에게만 보이게 하기

코드는 `/익명관리`를 관리자 기본 권한으로 제한하고, 실행 시 서버 소유자 여부를 다시 검사합니다. Discord 화면에서도 최대한 숨기려면:

1. 서버 설정 → **연동(Integrations)** → **리고스**
2. 명령어 목록에서 `/익명관리` 선택
3. `@everyone` 사용 권한을 끕니다.
4. 서버 소유자 본인만 사용자 예외로 추가합니다.

Discord 클라이언트·서버 설정에 따라 메뉴 이름은 조금 다를 수 있습니다. 이 설정을 하지 않아도 다른 관리자는 명령을 실행할 수 없지만, 명령어 이름이 목록에 보일 수 있습니다.

## 8. 기록되는 정보

- 숫자 메시지 ID
- 서버 ID
- 작성자 Discord ID
- 작성 당시 사용자명
- 원문
- `///` 변환 후 게시 내용
- 작성 채널
- 게시 채널
- 공개 메시지 ID
- 작성 시각

## 9. 기본 안전장치

- 외부 링크와 Discord 초대 링크 차단
- 모든 실제 멘션 비활성화
- 사용자별 기본 쿨타임 60초
- 기본 최대 길이 1,000자
- 명령 채널 제한
- 서버 소유자만 기록 조회 가능

## 10. 주의

이 봇은 다른 멤버에게만 익명입니다. 서버 소유자는 작성자를 확인할 수 있으므로, 서버 안내에 반드시 다음 취지의 문구를 적어 두세요.

```text
익명 메시지의 작성자 정보는 다른 멤버에게 공개되지 않습니다.
악용 방지와 운영을 위해 서버 소유자에게 작성자 ID, 원문 및 작성 시각이 기록됩니다.
```
